# Завдання №2

**Умова завдання:** виправити та дописати наявний код.

Для реалізації пропонуються два підходи:

---

### Підхід без інтерфейсу

Якби завдання полягало лише у створенні власного класу `CStrings` (без вимоги інтерфейсу), я б реалізував його на основі **шаблонного класу `CArray<T>`**.

**Переваги цього підходу:**
- Легко масштабувати на інші типи (`int`, `double` тощо).
- Не потрібно реалізовувати окремий інтерфейс або віртуальні методи.
- Контейнер повністю керує пам’яттю та операціями над елементами.

---

### Підхід з інтерфейсом

Якщо наявність інтерфейсу є важливою умовою завдання, реалізовано **шаблонний віртуальний інтерфейс `CArray2<T>`** та клас `CStrings2`, який його наслідує.

**Особливості цього підходу:**
- Використано **Implicit Template Specification** для забезпечення конкретної реалізації контейнера рядків (`std::string`).
- Забезпечує поліморфізм та можливість створювати інші класи-наступники без дублювання коду.
- Зменшує **code bloat**, характерний для повністю шаблонних рішень.
